Os requisitos deverão estar declarados de forma clara e direta,
para que não exista ambiquidades. Cada requisito tera sua indentificação
RFF = requisitos funcionais relacionados a interface
RFB = requisitos funcionais relacionados ao back-end
RN = requisitos nao funcionais
Cada requisito sera definico com o nível de importancia
S = Super importante
I = importante 
P = pouco importante
